## 3. Design decisions this checkpoint

1. **Variable registry as single source of truth (`scripts/env/schema.ts`).** Every variable is declared once — name, Zod schema, classification (`public`/`server`/`secret`), description, optional per-environment-class requirement. `validate.ts` builds its runtime checks entirely from this registry; nothing may be consumed without first being declared here (task §25).
2. **Environment-class enum sourced from architecture, not invented.** `ENVIRONMENT_CLASSES` is the verbatim seven-tier list from `docs/architecture/11_DEVOPS_WORKSTREAM.md` §2 — `local, development, testing, staging, uat, production, sandbox`. A new `CARGOGRID_ENV` variable carries this (distinct from Node's own 3-value `NODE_ENV` build-mode flag, which is also declared and validated separately).
3. **Symmetric production-linkage rule, override removed (deliberate strengthening vs. Prompt 85).** Prompt 85's script only checked "local, and provided an `ALLOW_NON_LOCAL_SUPABASE_URL` escape hatch. This task's rule is bidirectional and has no override: `local` must be loopback; every other tier must NOT be loopback (grounded in `11_*.md` §2's "every non-Local environment is a fully separate Supabase project" rule). Removing the override is a deliberate choice — Prompt 86 §24 ("no default secret or permissive fallback") reads more naturally without one, and a bypass would undermine exactly the safeguard this task exists to build. Recorded here explicitly rather than silently dropped.
4. **Static secret/public-prefix self-check, not a fabricated bundle scan.** No real bundler exists yet (`app/` is Phase 1 scope). `schema.ts` throws at module-load time if any `secret`-classified variable is declared with the `NEXT_PUBLIC_` prefix — a real, deterministic check grounded in Next.js's actual client-exposure mechanism. A genuine client-bundle content scan is explicitly deferred to `PH0-088`/Phase 1 CI, not faked here (`scripts/env/client-guard.ts` docstring records this).
5. **Zod adopted as the schema library — `ADR-0003`.** Chosen because `04_*.md`/`08_*.md` already name "Zod-or-equivalent" for the related (Phase 1) contract-validation concern; adopting it here establishes working precedent without binding Phase 1's own future ADR.
6. **`node:test` for this task's test suite, not a project-wide framework choice.** No test framework is chosen yet (`ADR-CAND-ARCH-022`, owned by `PH0-091`/Testing Foundation). `node:test` is Node's zero-dependency built-in — a real, working choice for this Phase 0 checkpoint's own tests, explicitly flagged in `validate.test.ts`'s docstring as provisional and not a preemption of `PH0-091`'s decision.
