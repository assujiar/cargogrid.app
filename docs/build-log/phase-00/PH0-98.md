## 8. Errors / recovery / risks / issues

One real defect found and fixed during authoring: `FlagCache`'s first draft used a TypeScript constructor parameter-property (`constructor(private readonly ttlMs: number) {}`), which `node --experimental-strip-types` cannot execute — that mode only erases type annotations, it does not perform TypeScript's parameter-property-to-field-assignment transform, so `pnpm run test` failed with `ERR_UNSUPPORTED_TYPESCRIPT_SYNTAX`. Fixed by declaring the field and assigning it explicitly in the constructor body — a real, disclosed authoring-time catch (this repository's toolchain choice, `ADR-0002`, has this exact constraint; no other file in the repository had used this syntax before, so this is the first time it was caught). No other error. No disclosed gap beyond §7's `NOT_RUN` table, all already-named and owned.
