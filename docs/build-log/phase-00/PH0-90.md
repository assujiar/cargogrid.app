## 2. Scope decision — documentation/ADRs only, no `components/ui/` code (disclosed, not silently narrowed)

Prompt 90 §20 task 2 asks to "implement approved tokens/theme/white-label primitives and core accessible components/states," and §28 wants "component tests/examples." Taken literally, this would require creating files under `components/ui/` and `lib/`. Both paths are explicitly **Phase 1 Platform Core** scope per `docs/architecture/04_REPOSITORY_TARGET_STRUCTURE.md`'s migration-wave table (wave 2: `app/(public)`, `app/(supreme)`, `lib/**`, `server/queries|mutations|actions/`, `server/policies/`, `server/jobs/`, **`components/ui/`**, `supabase/migrations/`) — the same boundary `ADR-0001` established for domain folders, and that `PH0-085` (toolchain), `PH0-086` (chose `scripts/env/` over `lib/config/` specifically because `lib/` is wave-2), and `PH0-089` (boundary rules configured inert until `lib/`/`server/` exist) all respected.

Creating real component files now — with no `app/` around them to consume, and no Phase-1-evidenced page requirement driving their exact shape — would be exactly the class of premature, unused scaffold `ADR-0001` exists to prevent, generalized from "domain folders" to "UI components with no consumer." **Decision: this checkpoint resolves the two ADR-required *decisions* (`ADR-CAND-ARCH-020/021`) and documents every structural convention Phase 1 needs, but creates zero `components/ui/`/`lib/`/token files.** This is the same pattern as `ADR-0002`/`ADR-0003` fixing the toolchain decision before `PH0-085` used it, applied here one layer further upstream (deciding before Phase 1 implements, not deciding *and* implementing prematurely).

## 3. Design decisions (`ADR-0005`, `ADR-0006`)

1. **Component foundation: Radix UI primitives (`radix-ui@1.6.2`, verified current via `npm view` this checkpoint), copy-in pattern into `components/ui/`.** Not a black-box runtime dependency — satisfies RPD-019 ("CargoGrid controls component structure") exactly, since 100% of what renders is CargoGrid's own copied-and-owned source.
2. **Design tokens: CSS custom properties (runtime, per-tenant override) authored via Tailwind v4's `@theme` CSS-first config (`tailwindcss@4.3.2`, verified current).** One token source, not two — v4's `@theme` block compiles directly to the same custom-property namespace consumed at runtime, closing the "two systems that could drift" risk `09_*.md` §13's original CSS-vs-Tailwind framing left open.
3. **Visual-regression/component-test tooling deliberately deferred to `PH0-091`.** `09_*.md` §13's own text calls this "a test-infrastructure choice, not a design decision" — resolving it under a design-system ADR would overreach into `PH0-091`'s (`ADR-CAND-ARCH-022`) scope. Recorded as an explicit deferral in `ADR-0006`'s propagation section, not silently dropped.
4. **CargoGrid's own base brand identity (primary color, logo, typography family) is an explicitly disclosed open item, not invented.** Verified this checkpoint (`grep` across all six blueprint documents for brand color/hex/logo/typography specification) that no such value exists anywhere in `docs/blueprint/**` — only the *tenant-configurable* white-label surface (Blueprint §18) is specified. Inventing a specific brand color here would be an unauthorized product/design decision, the same category `HANDOFF.md`'s prior `BLOCKED_DECISION` precedents refused to make unilaterally. Recorded in `DESIGN_SYSTEM.md` §3 with owner (Product/Design) and exact resolution point (before Phase 1's White-label Studio/Supreme Admin portal work, `PLT-117..119`, needs to render a default theme) — not blocking any nearer-term Phase 0/1 work.
5. **Reference values proposed for non-brand-specific token roles** (`neutral`, `success`, `warning`, `danger`, `info` — `DESIGN_SYSTEM.md` §2.1), chosen from well-established accessible color-scale conventions, explicitly labeled as *pending* the real contrast-validation gate `09_*.md` §9 requires (no automated contrast checker exists until Phase 1 implements it) — not claimed as WCAG-certified, since that certification cannot actually be run yet.
