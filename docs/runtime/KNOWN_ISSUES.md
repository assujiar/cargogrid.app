## 4. Issue records

### ISS-2026-002 — Parallel-session collision (no single-writer discipline)

Two sessions ran Prompt 21 concurrently on separate branches (`…-oanf5a`, `…-b492y3`); both merged to `main` and the merge corrupted the discovery baseline and duplicated the context set (see `ERROR_LEDGER.md` ERR-2026-001). Root cause: no lock/convention preventing concurrent execution of the same runtime step. **Handling:** designate one authoritative build branch per runtime step; do not launch parallel agent sessions on the same step; a future prompt must verify checkpoint lineage before writing. Status `OPEN` until a written single-writer convention is adopted (candidate: note in `AGENTS.md`).

**Recurrence (2026-07-14, third session):** a third branch, `claude/eloquent-mayer-s40hn4`, was cut from `main` at `d587445` *before* this reconciliation (`CG-S2-DISC-001-R1`) had merged. It independently discovered the same underlying corruption and — not knowing about `-R1` — resolved it the opposite way (root-level context kept canonical, `docs/runtime/*` marked superseded) while also completing Step 2 discovery Prompts 22–34. On merge with `main`, that branch's canonical-location choice was reverted in favor of this ratified decision (`docs/runtime/` canonical); its Step 2 discovery deliverables (Prompts 22–34, `docs/discovery/02_*.md`–`14_*.md`) were kept and layered on top of the reconciled baseline (see `TASK_LEDGER.md` `CG-S2-DISC-002..014` and `CHANGE_MANIFEST.md` `CHG-2026-003`). This is the second occurrence of this exact issue and reinforces that it remains `OPEN` until a single-writer convention is actually enforced, not just documented.

**Recurrence (2026-07-15, fourth occurrence — reconciliation never actually performed):** a fourth branch, `claude/sleepy-ride-4vxsk6`, diverged from `origin/main`@`27389a4` (PR #8, Prompt 45) and independently executed Prompts 46–51, Phase 0 kickoff (80), and Prompts 81–82 — nine commits, three full architecture documents, a full Phase 0 kickoff, and two Phase 0 capability prompts — in parallel with this branch (`agent/cargogrid-autonomous-build`) doing the equivalent Prompts 49–51/80/81. That branch opened GitHub PR #10 15 seconds after this branch's PR #9 merged into `main`, confirming near-simultaneous parallel execution. The two lineages' outputs for the same task IDs materially differ (e.g. 607 vs. 401 traced requirement items for the "same" Prompt 49/CG-S3-ARCH-014 output). A prior session halted further Phase 0 execution on discovering it and recorded `BLOCKED_WORKTREE` in `HANDOFF.md`, pending an explicit operator decision among three reconciliation options. Full detail: `ERROR_LEDGER.md` `ERR-2026-002`.

**Recurrence (2026-07-15, fifth occurrence — corruption now committed to `main`, escalated to Sev-1):** without that operator decision ever being recorded, PR #10 and then PR #11 (this run's designated branch) were both merged into `main` (`a8a197f`, `b7653cb`). Because neither merge produced git conflict markers (both lineages' edits were pure insertions relative to their shared base), git silently concatenated both divergent lineages' full content into the same files. `docs/architecture/14_REQUIREMENT_PHASE_TRACEABILITY.md`, `15_RISK_RANKED_CRITICAL_PATH.md`, and `16_STEP3_CLOSURE_REPORT.md` each now contain two complete, contradictory copies of the same document (confirmed: two `## 1.` sections in each, 607-item vs. 401-item traceability totals). `docs/runtime/HANDOFF.md`, `CARGOGRID_BUILD_STATUS.md`, and `TASK_LEDGER.md` show the same stacking pattern for narrative/status content. Full detail and evidence: `ERROR_LEDGER.md` `ERR-2026-003`. This is the clearest evidence yet that `ISS-2026-002` needs an *enforced* single-writer lock (e.g. a pre-flight check in each routine invocation that fails fast if any other open PR/branch already claims the current task-ID range, and a rule that merging a PR while a sibling PR covering the same task-ID range is still open requires explicit reconciliation, not a fast-forward/no-conflict merge) — a documented convention has now been silently violated five times.

**Resolution of the 5th-occurrence corruption (2026-07-15):** the operator authorized adopting **Lineage A** as authoritative. `docs/architecture/14..16_*.md` were rewritten as single coherent Lineage A documents (607-item baseline), Prompt 82 was re-verified against it, and the duplicate Phase 0 build logs were consolidated onto the singular prompt-package path. `ERR-2026-003` is `RECOVERED`. This cleared the *content* corruption and unblocked Phase 0.

**Enforcement adopted (2026-07-15, `CG-S5-PH0-008`, Prompt 87 — closes the *process* gap):** `docs/git/GIT_STRATEGY.md` §7 documents, and `AGENTS.md`'s "Required pre-flight" section now mandates, a two-part pre-flight collision check before starting any Phase 0+ prompt: (1) list open PRs/branches via the GitHub API/MCP and stop if another session claims the same task-ID range — **executed for real this checkpoint**: zero open PRs, and the only branch with commits ahead of `origin/main` is this session's own `agent/cargogrid-autonomous-build` (`claude/sleepy-ride-8pg1em`, this session's harness-assigned branch, is fully contained in `origin/main` — confirmed via `git merge-base`, not assumed); (2) `pnpm run git:check` (`scripts/git/check-worktree-collision.ts`, 4 passing tests) automates the locally-checkable half — a genuine fork between two `agent/*`/`claude/*` branches (each having commits the other lacks), the exact `ERR-2026-002`/`ERR-2026-003` precondition. **Status `RESOLVED`**: this is now a documented, followed, and partially-automated procedure rather than an unenforced convention silently skipped five times — no further action required unless a 6th occurrence demonstrates the procedure itself is insufficient.
